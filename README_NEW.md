# 🚀 Crypto OTC Dashboard Generator

## 📦 快速开始

### 方式一：文本输入（推荐！最简单）

**1. 运行文本输入工具：**
```bash
python3 generate_from_text.py
```

**2. 输入日期（可选）：**
```
请输入日期 (默认：2026-04-22): 
```
直接按回车使用当前日期，或输入自定义日期

**3. 粘贴数据（支持多种格式）：**
```
BTC 1200 150 进场期第 5 天
ETH 1300 180 进场期第 6 天
SOL 100 20 退场期第 3 天
```

**4. 输入 END 结束：**
```
END
```

**5. 自动生成 dashboard.html！**

---

### 方式二：编辑 JSON 文件

**1. 编辑 sample_data.json：**
```json
{
  "date": "2026-04-22",
  "coins": {
    "BTC": {
      "otc": 1200,
      "explosion": 150,
      "status": "进场期第 5 天"
    }
  }
}
```

**2. 运行生成脚本：**
```bash
python3 generate_from_sample.py
```

## 📝 数据格式说明

### 支持的输入格式：

**格式 1 - 空格分隔（推荐）：**
```
BTC 1200 150 进场期第 5 天
ETH 1300 180 进场期第 6 天
```

**格式 2 - 逗号分隔：**
```
BTC: 1200, 150, 进场期第 5 天
ETH: 1300, 180, 进场期第 6 天
```

**格式 3 - 竖线分隔：**
```
BTC | 1200 | 150 | 进场期第 5 天
ETH | 1300 | 180 | 进场期第 6 天
```

**格式 4 - 表格格式：**
```
币种    OTC     爆破    状态
BTC     1200    150     进场期第 5 天
ETH     1300    180     进场期第 6 天
```

## ❓ 常见问题

**Q: 为什么文本输入更方便？**  
A: 不需要手动编辑 JSON 文件，直接粘贴数据即可，支持多种格式自动识别。

**Q: 可以批量输入多天的数据吗？**  
A: 每次生成一个日期的图表。如需多日数据，可以：
1. 修改 `sample_data.json` 手动添加
2. 或者使用之前的 Notion 提取工具自动获取

**Q: 数据从哪里来？**  
A: 你可以：
- 从交易所/OTC 平台手动输入
- 从 Notion 页面提取（使用 `extract_data.py`）
- 从其他数据源导入

**Q: 如何部署到互联网？**  
A: 将生成的 `dashboard.html` 上传到：
- GitHub Pages
- Netlify
- 你的 Web 服务器
